| Signal               | Count | Status in v1.0.2          | Status in v1.0.3 (worktree, pre-fix) |

The `leading-toc-table` reduction validates the worktree's lane-agnostic T7/T8/T9 move. The remaining two patterns require code changes.

## Root Causes

### Empty-title

`any2md/heuristics.py:139` `refine_title` replaces a cover-page boilerplate H1 (`INTERNATIONAL STANDARD`, `WHITE PAPER`, etc.) with the first H2 found in the body:

```python
if refined.lower() in _COVER_PAGE_H1_VALUES:
    m = _H2_LINE_RE.search(body)
    if m:
        return m.group(1).strip()  # ← can be empty
```

Reproducer (verified):

```python
body = "# INTERNATIONAL STANDARD\n\n## \xa0\xa0\xa0\n"
refine_title("INTERNATIONAL STANDARD", body)  # → ""
```

The H2 regex `^##\s+(.+?)\s*$` allows `\s+` to span newlines, so a "## " line with no real content can capture stray characters that then strip to empty. Markdown-emphasis-only H2s (e.g., `## ***`) survive `.strip()` but produce a useless title.

`refine_title` also has a Wikipedia-prefix strip at `heuristics.py:185` that can leave `refined` empty when the input is the bare prefix (`Wikipedia:`).

### Orphan-punctuation

`any2md/pipeline/text.py:357` `strip_web_fragments` (T10) is registered only on the text lane (`text.py:475`). It performs two distinct removals:

1. **Orphan punctuation** — `^\s*[|>]+\s*$` lines (lone `|` or `>`). Universally bad output, regardless of extractor.
2. **Short incomplete sentences** — heuristic for trafilatura HTML extraction artifacts. Tuned for web noise; over-strips Docling's deliberate short headings/captions.

Docling's table parser occasionally emits malformed table rows containing only `|`. Without T10, those leak into the structured-lane output.

## Design

### Fix 1: `refine_title` walks all H2s (heuristics.py)

Replace the single-match H2 lookup with a finditer pass that returns the first H2 whose content is non-empty after stripping markdown emphasis. If none qualify, fall through to the original `refined`.

Wikipedia-prefix strip: only assign `refined` if the result is non-empty.

### Fix 2: Extract orphan-punct into its own stage (pipeline)

1. New helper in `any2md/pipeline/text.py`:
   ```python
   def strip_orphan_punctuation(text: str, options: "PipelineOptions") -> str:
       """Drop lone | or > lines. Lane-agnostic."""
   ```
2. `strip_web_fragments` (T10) calls `strip_orphan_punctuation` first, then runs the short-fragment heuristic on the result. Behavior on text lane unchanged.
3. `any2md/pipeline/structured.py` extends its lane-agnostic worktree block to include `strip_orphan_punctuation` alongside `strip_repeated_byline`, `dedupe_toc_table`, `strip_cover_artifacts`.

### Profile gating

Both fixes preserve current profile behavior:
- `refine_title` already runs unconditionally; the new walk doesn't change profile semantics.
- `strip_orphan_punctuation` is gated on `aggressive`/`maximum` profiles, mirroring the rest of T7/T8/T9/T10.
